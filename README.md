# step-io

STEP (ISO 10303) file I/O for Rust — read AP203 / AP214 / AP242 files
(geometry, assembly, and metadata) into a typed, in-memory model and write
them back.

> ⚠️ **Experimental.** Early and incomplete; the public API is not finalized
> and may change completely — expect breaking changes at any time.

## Design

The typed model (`StepModel`) and its read/write code are generated by the
`codegen` crate from a schema-faithful union of the AP203 / AP214 / AP242
EXPRESS schemas (`schema/universal.toml`):

1. **parser** — a Part 21 lexer + parser producing a raw entity graph
2. **read** — resolves the graph into the typed, arena-backed `StepModel`;
   non-standard input is normalized or dropped, with every entity accounted
   for in a `Report`
3. **write** — serialises a `StepModel` back to Part 21 text

Round-trip fidelity is the central goal, validated against a large corpus of
real-world CAD exports.
